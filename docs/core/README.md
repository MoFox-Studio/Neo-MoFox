# Core 层文档

本目录用于维护 `src/core` 的领域层文档。

## 目标

- 明确 core 层在三层架构中的职责边界
- 说明各模块的核心概念、对外能力和依赖关系
- 为后续模块级文档提供统一结构模板

## 模块导航

- [components](./components/): 组件定义、组件基类、组件装载与生命周期
- [config](./config/): core 层配置模型与初始化逻辑
- [managers](./managers/): 各类管理器的职责、协作与调用链
- [models](./models/): 领域模型、消息与流式上下文
- [prompt](./prompt/): 提示词模板、渲染策略与策略桶
- [transport](./transport/): 接收器、适配器、Sink、路由与分发链路
- [utils](./utils/): core 层通用工具函数与辅助能力

## 建议文档结构

每个模块目录建议至少包含以下章节：

1. 模块职责
2. 关键对象与数据结构
3. 调用流程
4. 对外接口
5. 与其他模块依赖关系
6. 常见问题与排查

## 编写约束

- 文档描述必须与当前实现一致，避免保留过时行为说明
- 涉及接口变更时，同步记录影响范围与迁移建议
- 保持与三层架构一致：kernel -> core -> app
